# PR Review Checklist

이 문서는 공개 PR을 받을 때 리뷰어가 같은 기준으로 확인하기 위한 체크리스트입니다.

## 1. 공개 경계

| 확인 항목 | 기준 |
|---|---|
| 실제 정보 없음 | 실고객, 실프로젝트, 연락처, 결제정보, 접속 정보가 없어야 한다. |
| 생성물 제외 | `output/`의 DOCX/XLSX/ZIP 파일은 포함하지 않는다. |
| 로컬 입력 제외 | `data/contract-input.json`, `data/*.local.json`, `data/*.private.json`은 포함하지 않는다. |
| 샘플 익명화 | 샘플은 재사용 가능한 가상 상황으로 작성한다. |
| 외부 글 처리 | 실무 글은 패턴만 일반화하고, 조항 출처처럼 직접 싣지 않는다. |

## 2. 계약 문구

| 확인 항목 | 기준 |
|---|---|
| 법률 자문처럼 보이지 않음 | 단정적 판단보다 초안, 검토, 분류, 체크 기준으로 쓴다. |
| 양측 효과 확인 | 발주자 이점, 수급자 이점, 균형안을 함께 고려한다. |
| 줄이는 분쟁 명시 | 새 문항이나 조항은 어떤 분쟁을 줄이는지 설명한다. |
| 견적 row 연결 | 금액, 전제조건, 완수 조건, 테스트 조건이 끊기지 않아야 한다. |
| 선택 조항 구분 | 항상 들어갈 조항과 선택 별첨을 섞지 않는다. |

## 3. 데이터와 생성기

| 확인 항목 | 기준 |
|---|---|
| 샘플 3종 유지 | 기본형, 디자인형, 운영형 입력이 모두 빌드되어야 한다. |
| 선택 별첨 작동 | 입력값에 따라 디자인/UX 별첨과 수정횟수 별첨이 포함 또는 제외되어야 한다. |
| 문서 수 확인 | 선택 별첨 포함 샘플은 19개 DOCX와 16개 XLSX, 제외 샘플은 17개 DOCX와 14개 XLSX가 생성되어야 한다. |
| 견적 ID 반영 | 견적 row ID가 RFP, 계약서, 검수기준표, 대금표, 견적표에 나타나야 한다. |
| DOCX/XLSX 무결성 | 생성된 DOCX, XLSX, ZIP은 압축 검사에 통과해야 한다. |

## 4. 필수 검증

```bash
npm run check
npm run audit:publish
```

계약 문구, 문서 빌더, 입력 샘플, 공개 경계를 바꾸는 PR은 위 명령을 통과해야 합니다.

## 5. 리뷰 결론

리뷰어는 PR에 아래 중 하나로 결론을 남깁니다.

| 결론 | 의미 |
|---|---|
| Approve | 공개 경계와 생성 검증을 통과했고, 문구 위험이 낮다. |
| Request changes | 공개 경계, 생성 검증, 계약 문구 중 수정이 필요하다. |
| Needs expert review | 제품 구조는 맞지만 전문가 검토가 필요한 문구가 있다. |

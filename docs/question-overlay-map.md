# 문항 오버레이 맵

이 문서는 입력 UI에서 각 문항 옆에 붙일 메모 기준이다. 사용자는 계약문서를 받지만, 시스템은 아래 기준으로 어떤 문항이 어떤 분쟁을 줄이는지 설명한다.

## 수급자 입력 흐름

외주를 받는 입장에서는 아래 순서로 입력한다.

1. `작성 관점`: `수급자` 선택
2. `착수자료`: PRD, 기능정의서, 화면설계서, 디자인 기준, 콘텐츠, 데이터/API, 검수기준의 제공 상태 선택
3. `기능 목록`: 기능 ID, 기능명, 완료 조건, 확인 방법, 제외 사항 입력
4. `견적 row`: 각 기능 또는 작업 묶음별 단위, 수량, 단가, 금액, 전제조건 입력
5. `제외 범위`: 계약금액에 포함되지 않는 작업 입력
6. `수정`: 무상수정 횟수, 수정 1회 기준, 경미한 수정 범위 선택
7. `변경요청`: 신규 기능, 신규 화면, 데이터 변경, 외부 연동 추가의 과금 단위와 착수 조건 선택
8. `회의`: 온라인 회의 주기, 대면회의 포함 횟수, 초과 비용, 회의록 승인 기준 선택
9. `대금`: 견적 row 합계와 연결되는 선금, 중도금, 잔금, 마일스톤 지급 기준 입력
10. `권리/인도`: 소스코드, 사전보유자산, 외부 라이브러리, 운영 계정 인도 범위 입력

## 문항별 오버레이

| 문항 | 입력 방식 | 오버레이 메모 | 줄이는 분쟁 | 계약문서 반영 |
|---|---|---|---|---|
| 작성 관점 | 버튼 | 수급자/발주자/중립에 따라 기본값과 강조 조항이 달라짐 | 한쪽에 치우친 조항 주장 | 입력값 요약표, 계약서 본문 |
| PRD 제공 상태 | 셀렉트 | 확정된 기능부터 착수하도록 일정 기준을 잡음 | 자료 미제공 상태 착수, 일정 지연 책임 | 개발 착수 조건 조항 |
| 화면설계서/Figma | 셀렉트 | 화면과 상태가 확정된 범위만 구현 대상으로 봄 | “이 화면도 당연히 포함” 주장 | RFP/과업내용서, 검수기준표 |
| 디자인 기준 | 셀렉트 | 색상, 라운딩, 간격, 레퍼런스 기준을 고정함 | 주관적 디자인 불만, 반복 수정 | 수정 조항, 변경요청서 |
| 기능 ID | 표 | 견적, 검수, 하자, 변경요청을 같은 ID로 연결함 | 기능 누락, 검수 기준 불일치 | RFP, 검수기준표, 하자신고서 |
| 완료 조건 | 표 | 완료의 정의를 기능별로 고정함 | 납품 완료 여부 다툼 | 검수 조항, 검수기준표 |
| 확인 방법 | 표 | 화면 확인, 테스트 계정, 파일 제출 등 확인 방식을 정함 | 검수 방식 다툼 | 검수기준표 |
| 제외 사항 | 표/문장 | 계약금액에 포함되지 않는 작업을 명시함 | 추가 작업 무상 요구 | 과업 범위 조항, 변경요청서 |
| 견적 row | 표 | 기능/작업 묶음별 단위, 수량, 단가, 전제조건, 완수 조건, 테스트 조건, 부수 항목을 남김 | 총액에 모든 작업이 포함됐다는 주장 | 견적산정표, 대금 지급표 |
| 연결 요구사항 | 표 | 견적 row를 기능 ID와 연결함 | 견적 항목과 실제 과업 불일치 | 견적산정표, RFP |
| 견적 전제조건 | 표 | 자료 제공, 승인, 외부 API 준비 등 전제조건을 둠 | 견적 범위 확대, 일정 지연 책임 | 견적산정표, 착수 조건 조항 |
| 견적 완수 조건 | 표 | row별 완료 기준을 발주자와 수급자가 같이 적음 | 완료 여부 다툼 | RFP, 계약서, 검수기준표 |
| 견적 테스트 조건 | 표 | row별 확인 방식과 테스트 시나리오를 둠 | 검수 방식 다툼 | 검수기준표, 하자신고서 |
| 견적 부수 항목 | 표 | 큰 row 안에 포함되는 세부 작업을 적음 | 세부 작업 포함 여부 다툼 | RFP, 계약서 과업 범위 조항 |
| 무상수정 횟수 | 숫자/셀렉트 | 전체 총량 또는 마일스톤별 횟수를 정함 | 기간 내 무제한 수정 | 하자와 무상수정 조항 |
| 수정 1회 기준 | 셀렉트 | 통합 수정요청 1세트를 1회로 봄 | 쪼개기 피드백, 반복 번복 | 수정요청서 |
| 하자 정의 | 셀렉트/문장 | 계약 범위 기능 불능 또는 재현 가능한 오류로 한정함 | 하자와 개선요청 혼동 | 하자 조항, 하자신고서 |
| 변경요청 대상 | 체크 | 신규 기능, 신규 화면, 데이터 구조 변경 등을 분리함 | 추가개발 무상 요구 | 변경요청 조항 |
| 변경요청 단가 | 표 | 최소 과금 단위와 단가를 미리 둠 | 추가 작업 비용 다툼 | 견적산정표, 변경요청서 |
| 온라인 회의 주기 | 셀렉트 | 정기 커뮤니케이션 범위를 정함 | 상시 대기 요구 | 회의 조항 |
| 대면회의 포함 횟수 | 숫자/셀렉트 | 포함 횟수와 1회 기준 시간을 정함 | 무제한 대면 미팅 요구 | 회의 조항, 합의표 |
| 회의록 승인 | 셀렉트 | 회의 결정사항의 효력 조건을 둠 | 구두 합의 다툼 | 회의록 승인서 |
| 지급 마일스톤 | 표 | 산출물, 검수, 지급을 연결함 | 납품했는데 대금 미지급 | 대금 지급표 |
| 소스코드 인도 | 셀렉트 | 인도 방식과 범위를 정함 | 결과물 권리와 유지보수 다툼 | 권리귀속/소스코드 인도목록 |
| 사전보유자산 | 표 | 수급자가 원래 가진 모듈과 자동화 자산을 분리함 | 모든 코드 양도 요구 | 권리귀속 조항 |
| 운영비 부담 | 셀렉트 | 서버, API, 문자, 도메인, 라이선스 비용 주체를 정함 | 운영비 포함 여부 다툼 | 운영비/API/계정 인수인계서 |

# 제품 브리프

## 이름

`scopeguard-kr`

## 목적

소송 자동화가 아니라, SW 외주개발계약에서 소송으로 가기 전 분쟁을 작게 만드는 계약서를 생성하는 것.

## 첫 범위

1. 계약 전 과업 범위를 선택형 입력값으로 구조화한다.
2. 입력값으로 분쟁 예방 기준이 반영된 HWP/XLSX 계약문서 초안을 만든다.
3. 견적 row를 기능 ID와 연결하고, 전제조건·완수 조건·테스트 조건·부수 항목을 계약 표의 기준으로 쓴다.
4. 체크리스트는 내부 점검 룰셋으로 두고, 결과를 계약서 조항과 부속서류에 반영한다.

상세 범위는 `docs/prd.md`를 기준으로 한다.

## 하지 않는 것

- 판결 예측
- 자동 법률 자문
- 변호사 대체
- 모든 업종 계약 자동화
- 분쟁 승패 판단

## 사용자

- 소규모 개발사
- 프리랜서 개발자
- 초기 스타트업
- 학교/연구실/기관과 계약하는 수급자
- SW 외주를 처음 발주하는 발주자
- 계약서 검토를 보조받고 싶은 변호사/PM

## 핵심 문장

> “가이드라인을 읽으라고 던지는 게 아니라, 그 기준이 반영된 계약서를 만들어준다.”

## 사업화 방향

오픈 저장소는 문서 템플릿과 작성 도구를 공개하고, 실제 사업화는 다음 데이터와 운영 레이어에서 한다.

- 익명화된 계약 리스크 패턴
- 업종별 RFP 점검 룰셋
- 계약서 조항 작성 룰셋
- 견적 row와 요구사항 연결 데이터
- 견적 row 입력 로그와 누락 패턴 분석
- HWP/PDF 문서 처리
- 변호사 검토 워크플로우

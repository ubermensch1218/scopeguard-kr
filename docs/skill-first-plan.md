# Skill-first Plan

초기 제품은 웹앱보다 Claude/Codex 스킬을 먼저 둔다. 웹은 나중에 입력 보조 UI로 붙인다.

## 이유

- 계약서 작성은 단일 폼으로 끝나지 않는다.
- 항목마다 값의 단위가 다르다.
- `0회`, `주간`, `월간`, `마일스톤별`, `최소 포함 횟수`, `초과 단가`, `N영업일`을 각각 물어야 한다.
- 발주자와 수급자 입장에 따라 기본값과 강조점이 달라진다.

## 스킬 우선 흐름

1. 작성 관점 선택: 발주자 / 수급자 / 중립
2. 버튼, 셀렉트, 숫자 입력, 표 입력으로 계약 파라미터 작성
3. 기능 ID와 견적 row 연결
4. 견적 row별 전제조건, 완수 조건, 테스트 조건, 부수 항목 입력
5. 필수 입력값과 완료 기준 누락 여부 확인
6. 문항별 오버레이 메모로 줄어드는 분쟁 표시
7. 표 형태 DOCX 계약문서 패키지 생성
8. 계약서 초안과 부속서류 검토
9. 참고문헌 링크 확인

## 포함 경로

- Codex: `.codex/skills/sw-contract-scopeguard/SKILL.md`

## 웹앱의 역할

웹앱은 최종 제품이 아니라 입력 보조 화면이다.

- 파라미터 선택 UI
- 내부 점검 룰셋 기반 누락 항목 표시
- DOCX 미리보기
- JSON 또는 ZIP 다운로드

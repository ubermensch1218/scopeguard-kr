# Document Writing Recipes

각 문서는 입력값, 산출 형식, 검증 기준을 가진다. 사용자가 특정 문서를 요청하면 해당 행의 입력을 먼저 채우고, 빠진 값은 `확인 필요`로 둔다.

Detailed per-document modules live under `references/documents/{문서번호-이름}/recipe.md`. Use this file as the summary index only when multiple documents are being planned together.

## 공통 작성 규칙

- DOCX는 사람이 읽고 승인하는 문서로 쓴다.
- XLSX는 행 단위 체크, 긴 목록, 검수 실행, 리스크 검토에 쓴다.
- 요구사항 ID, 견적 row ID, 검수 기준, 완료 기준, 테스트 조건을 가능한 한 함께 연결한다.
- 신규 기능, 신규 화면, 데이터 구조 변경, 승인사항 번복은 수정요청이 아니라 변경요청으로 분류한다.
- 하자는 계약 범위 기능 불능 또는 검수 기준 미달로 한정한다.
- 모호한 값은 임의 확정하지 말고 `확인 필요`로 표시한다.

## 기능별 구현·디자인 명세 작성 순서

1. 기능 ID와 연결 견적 row를 정한다.
2. 진입점을 화면, 버튼, URL, 메뉴, 알림 단위로 적는다.
3. 사용자/권한을 적는다.
4. 기본, 로딩, 빈 상태, 오류, 권한 없음, 완료 화면을 나눈다.
5. Mermaid 플로우 원문을 작성한다.
6. 최종 화면과 최종 결과를 구분한다.
7. 완료 기준과 테스트 조건을 검수기준표에 연결한다.

## 입증 기준

- 스킬은 이 파일을 기준으로 문서별 필수 입력을 누락 없이 물어야 한다.
- 생성기는 DOCX/XLSX 파일을 만들고 압축 무결성 검사를 통과해야 한다.
- 공개 감사는 각 파일의 필수 문구와 시트 내용을 확인해야 한다.

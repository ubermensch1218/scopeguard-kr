# 공급자/수요자 합의 옵션표

이 표는 어느 한쪽을 방어하기 위한 문서가 아니라, 계약 전에 서로 예측 가능한 기준을 고르는 문서입니다.

## 1. 착수자료

| 옵션 | 수요자 이점 | 공급자 이점 | 권장 상황 |
|---|---|---|---|
| 기능별 착수자료 확정 후 착수 | 원하는 기능 누락 방지 | 불명확한 요구 착수 방지 | 기본 권장 |
| 전체 PRD 확정 후 일괄 착수 | 전체 그림 고정 | 범위 통제 강함 | 중형 이상 프로젝트 |
| 일부 기능만 우선 착수 | 빠른 시작 가능 | 확정된 범위만 책임 | 소액/빠른 MVP |

예시 문구:

```text
수급자는 PRD, 기능정의서, 화면설계서, 검수기준이 서면 확정된 기능에 한하여 개발 착수 의무를 부담한다.
```

## 2. 무상수정

| 옵션 | 수요자 이점 | 공급자 이점 | 권장 상황 |
|---|---|---|---|
| 전체 프로젝트 총 2회 | 납품 후 피드백권 확보 | 총량 통제 | 기본 권장 |
| 마일스톤별 1회 | 단계별 검토 가능 | 단계별 종료 명확 | 범위가 큰 프로젝트 |
| 무상수정 없음, 하자만 보완 | 비용 명확 | 수정 리스크 최소 | 저가/단기 작업 |

권장 문구:

```text
수정 1회는 산출물 제출 후 5영업일 이내 서면 제출된 통합 수정요청 1세트를 의미한다.
```

## 3. 하자

| 옵션 | 수요자 이점 | 공급자 이점 | 권장 상황 |
|---|---|---|---|
| 검수기준 미달은 무상 보완 | 계약한 기능 보장 | 하자 범위 명확 | 기본 권장 |
| 재현 절차 필수 | 오류 확인 가능 | 모호한 불만 차단 | 모든 프로젝트 |

권장 문구:

```text
계약 범위 기능이 작동하지 않거나 재현 가능한 오류로 검수기준을 충족하지 못하는 경우 하자로 보며, 무상수정 횟수에 산입하지 않는다.
```

## 4. 변경요청

| 옵션 | 수요자 이점 | 공급자 이점 | 권장 상황 |
|---|---|---|---|
| CR 승인 후 착수 | 필요 변경 가능 | 공짜 범위 확장 방지 | 기본 권장 |
| 시간 단가 기준 | 작은 변경 빠른 처리 | 산정 명확 | 스타트업/소액 |
| 별도 견적 기준 | 큰 변경 정밀 산정 | 일정/비용 통제 | 중형 이상 |

권장 문구:

```text
신규 기능, 신규 화면, 데이터 구조 변경, 외부 연동 추가, 승인된 PRD 또는 화면설계서 변경은 변경요청으로 처리한다.
```

## 5. 회의

| 옵션 | 수요자 이점 | 공급자 이점 | 권장 상황 |
|---|---|---|---|
| 대면 2회 포함 | 주요 의사결정 가능 | 회의 총량 통제 | 기본 권장 |
| 온라인 정기회의 포함 | 진행상황 확인 | 이동시간 절감 | 대부분 |
| 추가 대면 유상 | 필요 시 확장 가능 | 출장/PM 비용 보전 | 현장 요구 프로젝트 |

권장 문구:

```text
회의에서 논의된 사항은 회의록으로 등록되고 양 당사자가 승인한 경우에만 계약상 확정사항으로 본다.
```

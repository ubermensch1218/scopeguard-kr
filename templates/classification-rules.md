# 하자·무상수정·변경요청 분류 기준

## 하자

계약 범위에 명시된 기능이 작동하지 않거나, 재현 가능한 오류로 검수 기준을 충족하지 못하는 경우입니다. 하자는 무상수정 횟수에서 차감하지 않습니다.

예시:

- 저장 버튼 클릭 시 저장되지 않음
- 로그인 기능이 검수 기준대로 동작하지 않음
- PDF 다운로드가 계약 범위에 있는데 파일이 생성되지 않음

## 경미한 수정

계약 범위 내 산출물의 본질적 구조를 바꾸지 않는 문구, 오탈자, 버튼명, 이미지 교체, 색상, 라운딩, 간격 조정입니다. 무상수정 횟수 내에서 처리합니다.

예시:

- 버튼명을 “저장”에서 “제출”로 변경
- 오탈자 수정
- 라운딩 8px에서 12px로 조정

## 변경요청

승인된 PRD, 화면설계서, 기능정의서, 디자인 방향, 데이터 구조, 운영 정책, 외부 연동, 신규 화면 또는 신규 기능을 바꾸는 요청입니다. 별도 비용과 일정 합의가 필요합니다.

예시:

- PDF 다운로드만 계약 범위인데 엑셀 다운로드 추가
- 학생 화면만 있었는데 관리자 통계 화면 추가
- 기존 DB 구조를 변경해야 하는 기능 추가
- 승인된 Figma 화면 흐름 변경
